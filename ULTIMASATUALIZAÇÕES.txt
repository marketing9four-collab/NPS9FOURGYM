O QUE JA FOI FEITO (ordem cronologica resumida)
-------------------------------------------------
1. App completo criado do zero: Next.js + TypeScript + Tailwind + Supabase.
2. Encuesta publica em espanhol mexicano (ES-MX):
   - Tela de boas-vindas (fundo preto, texto branco, logo, ~2.5s, depois some).
   - Banner "banner_1.png" no topo.
   - 10 perguntas: 8 de avaliacao 0-10 (recepcion, equipos, "Higiene, limpieza
     y organizacion de nuestras instalaciones", banos, "Clases grupales y
     spinning", entrenadores, gerencia, estacionamiento), 1 pergunta NPS
     ("¿Que tan probable es que nos recomiendes...?"), e 1 pergunta de
     multipla escolha ("¿Que red social utilizas mas?" — TikTok, Instagram,
     Facebook, YouTube, Kwai, Twitter (X), Pinterest, No utilizo, Otra c/
     campo de texto livre).
   - Botao "Quiero comentar esto" em cada pergunta de nota (comentario opcional).
   - Etapa de contato opcional (nome/email) com envio anonimo.
   - Tela de confirmacao final.
   - Layout mobile: botoes de nota quebram em varias linhas (sem precisar
     arrastar), graficos e tabelas com scroll horizontal quando necessario.
3. ENCUESTA DIVIDIDA POR UNIDADE (21/07): antes tinha um seletor de unidade
   dentro da propria pagina; agora sao 4 rotas fixas (/arboledas, /metepec,
   /interlomas, /lomas), cada uma ja envia a unidade certa automaticamente,
   sem seletor. Continua tudo no MESMO banco e MESMO painel admin.
4. Painel admin (/admin/nps-9fourgym) 100% em PORTUGUES (PT-BR), conforme
   pedido, mesmo com a encuesta publica em espanhol:
   - Login protegido (cookie assinada, senha com hash bcrypt).
   - Metricas gerais, graficos (media por pergunta, comparacao por unidade,
     distribuicao de notas, distribuicao da pergunta de rede social),
     tabela de respostas, filtros (unidade/data/busca), paginacao,
     exportar CSV, botao "Apagar todos os dados" (com confirmacao).
5. Banco de dados (Supabase):
   - Tabela survey_responses com RLS (publico so insere).
   - Campo answers (jsonb, aceita numero 0-10 ou texto pra pergunta de
     escolha), comments (jsonb), overall_average calculado ignorando N/A e
     ignorando a pergunta de rede social (que nao e numerica).
6. Deploy: GitHub -> Vercel, com todas as env vars configuradas.

COMO RETOMAR O TRABALHO
--------------------------
1. Abrir este projeto de novo com o Claude Code.
2. Rodar "npm run dev" para testar local (precisa do .env.local configurado,
   ja esta).
3. Qualquer alteracao: editar o codigo, rodar "npx tsc --noEmit" para
   verificar erros, depois "git add -A", "git commit -m ...",
   "git push origin main", e "vercel --prod --yes" para publicar.
4. CUIDADO: o repositorio do GitHub e PUBLICO. Nunca commitar senhas, chaves
   ou o arquivo de links/credenciais sem estar no .gitignore.
