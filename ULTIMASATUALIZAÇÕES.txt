O QUE JA FOI FEITO (ordem cronologica resumida)
-------------------------------------------------
1. App completo criado do zero: Next.js + TypeScript + Tailwind + Supabase.
2. Encuesta publica em espanhol mexicano (ES-MX):
   - Tela de boas-vindas (fundo preto, texto branco, logo, ~2.5s, depois some).
   - Banner "banner_1.png" no topo.
   - Selecao de unidade (Arboledas, Metepec, Interlomas, Lomas).
   - 8 perguntas de avaliacao (0-10 + N/A): recepcion, equipos, limpieza,
     banos, clases_colectivas, entrenadores, gerencia, estacionamiento.
   - Botao "Quiero comentar esto" em cada pergunta (comentario opcional).
   - Etapa de contato opcional (nome/email) com envio anonimo.
   - Tela de confirmacao final.
3. Painel admin (/admin/nps-9fourgym) 100% em PORTUGUES (PT-BR), conforme
   pedido, mesmo com a encuesta publica em espanhol:
   - Login protegido (cookie assinada, senha com hash bcrypt).
   - Metricas gerais, graficos (media por pergunta, comparacao por unidade,
     distribuicao de notas), tabela de respostas, filtros (unidade/data/busca),
     paginacao, exportar CSV, botao "Apagar todos os dados" (com confirmacao).
4. Banco de dados (Supabase):
   - Tabela survey_responses com RLS (publico so insere).
   - Campo answers (jsonb), comments (jsonb), overall_average calculado
     ignorando N/A.
5. Deploy: GitHub -> Vercel, com todas as env vars configuradas.


BUGS ENCONTRADOS E CORRIGIDOS
-------------------------------
1. Hash da senha do admin com "$" sendo expandido pelo Next.js em .env.local
   -> corrigido escapando com "\$" (so no .env.local local; no Vercel nao
   precisa escapar).
2. Middleware do Vercel nao resolvia import "@/lib/auth/session" (Edge
   Function bundling) -> corrigido usando import relativo "./lib/auth/session".
3. BOM (caractere invisivel U+FEFF) entrando nas variaveis de ambiente do
   Supabase ao configurar via PowerShell -> corrigido com .trim() em todos os
   lugares que leem env vars (lib/env.ts, lib/auth/session.ts,
   lib/supabase/anon.ts, lib/supabase/server.ts).
4. Login admin falhando com usuario/senha corretos -> mesmo motivo do BOM,
   corrigido com o .trim().

COMO RETOMAR O TRABALHO
--------------------------
1. Abrir este projeto de novo com o Claude Code.
2. Rodar "npm run dev" para testar local (precisa do .env.local configurado,
   ja esta).
3. Qualquer alteracao: editar o codigo, rodar "npx tsc --noEmit" para
   verificar erros, depois "git add -A", "git commit -m ...",
   "git push origin main", e "vercel --prod --yes" para publicar.
